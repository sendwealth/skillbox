# 知识库降级策略：当外部搜索全部被封时的 Phase A 执行方案

## 触发条件

当 Phase A（获取情报）中所有搜索引擎和平台均不可用时（Google→验证码、Bing→低质量结果、搜狗微信→反爬、知乎→空白页、公众号→验证码）：

## 降级执行流程

### 1. 快速探测（3次以内）
- 尝试导航到搜狗微信搜索 (weixin.sogou.com)
- 尝试知乎搜索
- 尝试一两个行业垂直平台（如人人都是产品经理、鸟哥笔记）
- 如果全部失败 → 触发降级

### 2. 知识驱动调研模式
不再依赖实时搜索，而是：

**Step A: 结构化知识提取**
- 基于对目标领域的体系化训练知识
- 按照 OSCAR Phase C 的清单逐项输出分析
- 每个结论标注置信度：✅高（行业共识）/ 💬中（基于推理）/ ❓低（需验证）

**Step B: 定向验证（轻量）**
- 用 HN Algolia API (hn.algolia.com/api/v1/search) 搜英文相关讨论
- 导航到特定已知网站验证公司/产品是否存在
- 用 browser_navigate 访问 Wikipedia 补充事实

**Step C: 置信度透明**
- 报告中添加「置信度 & 局限说明」章节
- 列出需要用户自行验证的信息清单
- 标注信息来源：训练知识 / 公开验证 / 推理

### 3. 输出格式
调研报告仍然使用完整的 OSCAR Phase R 模板，但增加：
- 第八节：置信度 & 局限说明
- 第九节：需要交叉验证的信息（具体操作指南给用户）

## 不要做的事
- 不要反复尝试不同的搜索引擎（浪费时间）
- 不要对用户隐瞒搜索失败的事实
- 不要在缺少实时数据时编造具体数字
- 不要让子代理去爬（它们也会被同样的封锁卡住）
